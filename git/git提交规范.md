# git commit 提交规范
在使用git管理代码时，常常会随心所欲的提交各类注释，导致下次辨识度太差，而且不利于长期维护，一直想要有一个好的处理方法。曾经想通过，一个类似的前缀来标识
```
fix  修复
bug  修改
build 打包
test  测试
...
```
通过运行 `git commit -m fix ...` 表示此次是进行代码修复。。。
